# 🔄 세션 핸드오버 워크플로우

> **목표**: 현재 상태를 명확히 기록하고, 이전 세션의 역사를 보존하면서 문서를 가볍게 유지하기 위함. (스냅샷 + 아카이브 방식)

## 1. 핸드오버 트리거 확인

- **스텝 350~450 도달 시** (AI 체력 고갈)
- 또는 사용자가 `/handover` 명령어 입력 시

## 2. 히스토리 아카이브 (스냅샷 저장)

1. `docs/history/` 폴더가 없으면 생성한다.
2. 현재 날짜/시간으로 파일명 생성 (예: `docs/history/session_YYYYMMDD_HHMM.md`)
3. 현재 `docs/HANDOVER.md`의 내용을 **그대로 복사**하여 위 파일명으로 저장한다.
   - (명령어: `cp docs/HANDOVER.md docs/history/session_...md`)

## 3. 새로운 핸드오버 작성 (백지화)

1. `docs/HANDOVER.md`를 **새로** 작성한다 (덮어쓰기).
2. 아래 템플릿을 사용하여 **상세하게** 작성한다.

### 템플릿

```markdown
# 🔄 세션 핸드오버

## 📌 [최신] 세션 N (YYYY-MM-DD)

### 🏗️ 아키텍처 및 핵심 맥락 (상세!)
> **다음 세션에게 보내는 편지**: 소스 코드를 일일이 읽지 않아도 이해할 수 있도록 상세하게 적으세요.
> - **구조 변경**: (예: `Login.js`를 `auth/` 폴더로 이동했음)
> - **주요 로직**: (예: 점수 계산 로직이 `ScoreManager`로 통합됨)
> - **주의사항**: (예: `gh` 명령어 쓰면 터미널 멈춤)

### ✅ 완료된 작업
1. 작업 1
2. 작업 2

### 🚧 진행 중 / 남은 작업
- [ ] 작업 A (이슈 #10)
- [ ] 작업 B
```

> **주의**: 이전 세션의 "남은 작업" 목록을 복사하지 말고, **현재 시점의 이슈(GitHub Issue)**를 기준으로 작성할 것.

## 4. 완료 보고

- "핸드오버 완료. 이전 기록은 `docs/history/...`에 저장됨." 메시지 출력.
