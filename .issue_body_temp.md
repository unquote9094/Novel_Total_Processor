## 🎓 배경 지식 (Context)

Novel Total Processor 프로젝트는 기존 Node.js 기반에서 Python으로 완전히 재구성됩니다. 이 작업은 프로젝트의 기반을 다지는 첫 단계로, Python 프로젝트 구조, 설정 파일, 로깅 시스템을 구축합니다.

## 🎯 목표 (Goal)

1. 기존 JS 파일 제거 (docs/ 폴더는 보존)
2. Python 프로젝트 구조 생성 (`src/`, `config/`, `data/`)
3. 의존성 관리 파일 작성 (`pyproject.toml`, `requirements.txt`)
4. 전역 로깅 시스템 구축 (파일 DEBUG, 콘솔 INFO)
5. 설정 파일 템플릿 생성 (`config.yml`, `rules.yml`, `epub_style.css`)

## 🛠️ 기술적 계획 (Technical Plan)

### 생성된 파일
- `pyproject.toml`: 프로젝트 메타데이터, 의존성, 빌드 설정
- `requirements.txt`: pip install용 의존성 목록
- `src/novel_total_processor/__init__.py`: 패키지 진입점
- `src/novel_total_processor/utils/logger.py`: 전역 로깅 모듈
  - 파일 로그: `data/logs/YYYY-MM-DD.log` (DEBUG 레벨)
  - 콘솔 로그: stdout (INFO 레벨)
- `config/config.yml`: 프로젝트 설정 (경로, API, 처리 옵션)
- `config/rules.yml`: 파일명 규칙 (제목/화수/상태/별점/장르/작가/태그)
- `config/epub_style.css`: EPUB CSS 템플릿
- `README.md`: 프로젝트 문서
- `.gitignore`: Python 프로젝트용

### 주요 의존성
- `google-generativeai`: Gemini API
- `perplexityai`: Perplexity API
- `ebooklib`: EPUB 생성/수정
- `rich`: TUI
- `typer`: CLI
- `xxhash`: 빠른 해싱
